Question: If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this?

Answer: Có, Java cho phép làm điều này thông qua một tính năng gọi là Constructor Overloading. Tuy nhiên, có một điều kiện quan trọng về mặt kỹ thuật:

1. Điều kiện để Java chấp nhận
Java phân biệt các hàm khởi tạo dựa trên kiểu dữ liệu và số lượng tham số, chứ không phải tên tham số.
- Hợp lệ: Nếu một cái là String (Title) và một cái là int (ID).
- Không hợp lệ: Nếu cả Title và Category đều là kiểu String. Java sẽ báo lỗi vì không biết đang gọi cái nào.


2. Ví dụ minh họa
Trường hợp hợp lệ (Khác kiểu dữ liệu hoặc số lượng):
public class DVD {
    // Constructor theo Title
    public DVD(String title) { ... }

    // Constructor theo Category và Year (Khác số lượng tham số)
    public DVD(String category, int year) { ... }
}

Trường hợp lỗi (Cùng kiểu String):
// Java sẽ báo lỗi vì cả hai đều là (String)
public DVD(String title) { ... }
public DVD(String category) { ... }


3. Giải pháp khi cùng kiểu dữ liệu
Nếu bắt buộc muốn tạo DVD chỉ bằng Title hoặc chỉ bằng Category (đều là String) thì nên dùng Static Factory Method:
public static DVD createByTitle(String title)
public static DVD createByCategory(String category)


Tóm lại: Java cho phép nạp chồng hàm khởi tạo, nhưng danh sách tham số của chúng phải khác nhau về cấu trúc để trình biên dịch phân biệt được.
